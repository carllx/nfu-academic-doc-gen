## 1. 教学制度要求

### 1.1 实验指导书
*   **对应关系**：必须与教学大纲中的实践学时、内容严格对应。
*   **学时一致**：所有实验学时总和 ≤ 大纲实践学时。

### 1.2 实验报告
*   **评分制**：4 级制（优/良/中/不及格）。
*   **实验类型要求**：不低于 3 种（演示性、验证性、综合性、设计性）。
    *   *演示性*：教师主导演示，学生观察记录。
    *   *验证性*：学生操作，结果有标准答案；通常跟着课程步骤实验已知效果。
    *   *综合性*：综合多门课或多章知识；通常为期末大作业（实验 4）。
    *   *设计性*：学生自行设计方案并实施；考察已知知识应用于创作。
*   **4 实验模型**（标准规范）：
    *   **实验 1-3**：对应平时作业，分别考核各核心模块知识点，计入平时成绩。
    *   **实验 4**：期末综合项目（综合/设计性），考核全流程综合能力，计入期末成绩。
    *   **成果要求**：实验 4 必须提交**项目作品**与**项目报告**两份材料。
*   **提交形式**：可小组提交（1-5 人）。
*   **归档份数**：依据指导书项目数和实际收取数。

### 1.3 评分规则
*   **分差控制**：平时作业与实践报告分差、期末作业分差原则上 ≤ 5 分。
*   **成绩对应**：实验报告评分（优/良/中/不及格）**必须**与期末/平时成绩单填写分数一致，参考教学大纲评分标准内容。

## 2. 数据源规范 (Data Source)

### 2.1 `experiments` 字段（`course.yaml`）

> **SSOT**：`scripts/course_schema.py` 中的 `ExperimentItem` 模型。以下为快速参考。

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `id` | Number | ✅ | 实验序号（1, 2, 3, 4…） |
| `name` | String | ✅ | 实验名称（对应大纲编号） |
| `type` | String | ✅ | 实验类型：验证性 / 综合性 / 设计性 / 演示性 |
| `hours` | Number | ✅ | 实验学时 |
| `group_size` | Number | ✅ | 每组人数（alias: `group`）|
| `requirement` | String | ✅ | 必做 / 选做 |
| `objectives` | String | ❌ | 实验目的（多行，`（一）（二）` 分项；用于指导书「一、实验目的」及报告封面「一、实验目的」节）|
| `equipment` | String | ❌ | 实验设备与环境（多行，通常「硬件+软件」2 行；用于「二、实验设备与环境」/「二、实验条件（环境）」节）|
| `requirements` | String | ❌ | 实验要求（多行，`（一）（二）` 分项；用于「三、实验要求」节）|
| `methods` | String | ❌ | 实验步骤与要点 / 实验方法（原理）（多行；验证/演示性含详细步骤+图示；设计/综合性含创作框架；用于「四、实验步骤与要点」及「三、实验方法（原理）」节）|
| `summary` | String | ❌ | 实验内容简介（1-3 句；用于认定表「实验内容简介」单元格）|
| `conclusions` | String | ❌ | 实验结论（多行；验证/演示性含实验原理+数据表格；综合/设计性含作品说明；用于「五、实验结论」节）|
| `questions` | String | ❌ | 思考题（多行，`（一）（二）` 分项，2-4道；**仅**指导书「六、思考题」节，实验1-2适用，实验3、综合项目无此节）|

> **字段来源说明**：`objectives`/`methods`/`equipment` 对应过往项目 `course_config.yaml` 里的 `experiment_objectives`/`experiment_methods`/`experiment_equipment`，本项目统一简化字段名。`requirements`/`conclusions`/`questions`/`summary` 为本项目新增字段。无此字段时，对应节保留空白占位行供教师手填。

> **`summary` 字段说明**：仅用于认定表（Template 2），对应注释"实验内容简介"。内容应为 1-3 句概括性的描述，区别于 `objectives` 的目的描述。

```yaml
# course.yaml 示例
experiments:
  - id: 1
    name: "交互产品设计（一）"
    type: "验证性"
    hours: 4
    group_size: 1
    requirement: "必做"
    objectives: "掌握 Figma 自动布局与组件功能，完成基础交互原型搭建。"
    methods: "规范化制作：通过 Figma + VS Code 实现代码级交互开发。"
    equipment: |
      1. 硬件：个人计算机（学校机房或自备）
      2. 软件：Windows 10+；Figma、VS Code
  - id: 4
    name: "综合设计项目"
    type: "设计性"
    hours: 8
    group_size: 3
    requirement: "必做"
    summary: "学生自选课题，综合运用交互设计全流程完成一份完整的产品原型及报告。"
```

## 4. 生成器规范

### 4.1 入口与输出

*   **脚本路径**：`scripts/gen_experiment_xml.py`
*   **统一入口函数**：`gen_experiment(base_dir, output_dir, context)`

#### Template 2 — 实验认定表（`Template_Exp_Recognition.docx`）

*   **触发条件**：`experiments[i].type` 为 `综合性` 或 `设计性` 时生成。**每个符合类型的实验均生成一份认定表，不限于实验4**（即：验证性/演示性实验不触发，综合性/设计性实验每项各触发1份）。
*   **勾选框机制**：模板中以文本括号 `（   ）` 模拟勾选框——**不是** OOXML `<w14:checkbox>` 控件。脚本替换方式：
    ```python
    # 对应类型替换为已勾选，其余保留空格
    text = text.replace("（   ）", "（ √ ）")  # 仅替换当前类型对应的行
    ```
*   **各课程认定表预期数量**：
    *   **信息可视化**：实验2（设计性）、实验3（综合性）、实验4（综合性）→ **3 份**
    *   **交互产品开发**：实验2（综合性）、实验3（设计性）、实验4（综合性）→ **3 份**
*   **注意**：认定表不填写「学院」字段，仅填写「开课专业」（`course.major`），二者区分清楚。

| 占位符标签 | 注释说明 | 数据源 |
|-----------|---------|--------|
| 开课专业 | 例如：数字媒体艺术；不需要带上年级 | `course.major` |
| 项目名称 | 根据指导书实验项目一览表内容撰写，必须一致 | `experiments[i].name` |
| 学时 | 对应实验一览表学时，必须一致 | `experiments[i].hours` |
| 综合性实验（ √/空）| 是否综合性实验；是→`√`，否→`' '` | `experiments[i].type == '综合性'` |
| 设计性实验（ √/空）| 是否设计性实验；是→`√`，否→`' '` | `experiments[i].type == '设计性'` |
| 实验内容简介 | 1-3句概括性描述 | `experiments[i].summary`（若有）|
| 所属课程名称 | — | `course.name` |
| 专家认定意见 | **必须手写**，内容固定："符合实验要求，予以认定。" | ❌ **留空** |
| 专家名单（签名）| 须副教授/教授手签，**不可**签章或代签 | ❌ **留空** |
| 教学单位意见 | **必须手写**，内容固定："同意。" | ❌ **留空** |
| 教学单位负责人签字 | 冯原院长签章 | ❌ **留空** |
| 日期 | 开学第一周内均可（示例：2026年3月2日）| 从 `semester` 推断开学日期 |

#### Template 3 — 实验指导书（`Template_Exp_Guide.docx`）

**封面字段：**

| 占位符标签 | 注释说明 | 数据源 |
|-----------|---------|--------|
| 课程代码 | 与教学大纲一致 | `course.code` |
| 课程名称 | 与教学大纲一致 | `course.name` |
| 适用年级 | 从开课班级推断 | 从 `classes` 字段推断（示例：2023级）|
| 适用专业 | — | `course.major` |
| 日期（封面下方） | 年份，格式：二零XX年X月 | 从 `semester` 解析 |

**目录（TOC）字段：**

| 占位符 | 数据源 | 说明 |
|--------|--------|------|
| 实验一 XXXX | `experiments[0].name` | 目录第1条 |
| 实验二 XXXX | `experiments[1].name` | 目录第2条 |
| 实验三 XXXX | `experiments[2].name` | 目录第3条 |
| 综合项目 XXXX | `experiments[3].name` | 目录第4条 |

**一览表字段（每实验一行）：**

| 列名 | 数据源 |
|------|--------|
| 序号 | `experiments[i].id`（数字） |
| 项目名称 | `experiments[i].name` |
| 实验类型 | `experiments[i].type` |
| 开出要求 | `experiments[i].requirement`（必做/选做）|
| 每组人数 | `experiments[i].group_size` |
| 学时 | `experiments[i].hours` |
| 合计行（项目数）| `len(experiments)` |
| 合计行（学时）| `sum(e.hours for e in experiments)` |

**每节实验正文**（4 节：实验一/二/三 + 综合项目，按 `experiments` 列表顺序）：

> **实验节标题格式**：
> - 实验 1-3：`实验一/二/三 XXXX`（中文序数 + 实验名称）
> - 实验 4（综合项目）：`综合项目 XXXX`（固定前缀，不用「实验四」）

| 节标题 | 数据字段 | 适用实验 |
|--------|---------|----------|
| 一、实验目的 | `objectives` | 实验 1-4 全部 |
| 二、实验设备与环境 | `equipment` | 实验 1-4 全部 |
| 三、实验要求 | `requirements` | 实验 1-4 全部 |
| 四、实验步骤与要点 | `methods` | 实验 1-4 全部 |
| 五、实验结论 | `conclusions` | 实验 1-4 全部 |
| **六、思考题** | `questions` | **仅实验 1-2**；实验 3 和综合项目**无此节** |

*   **多行文本**：使用 `fill_multiline()` 逐段填充，保留原始 `pPr`。
*   **无数据时**：保留空白 XXXX 占位段落，供教师后续手填。
*   **章节模板克隆**：使用 `copy.deepcopy()` 克隆首节结构，动态生成后续节，**不手动构造 XML 格式属性**。

### 4.3 输出命名规范

```
output/实验材料/
├── {课程名}_实验报告封面_实验{N}.docx          # Template 1 × N 个实验（单班）
├── {课程名}_实验报告封面_实验{N}_{班级}.docx   # Template 1 × N × 班数（多班，如影视班/游戏班）
├── {课程名}_实验认定表_实验{N}.docx            # Template 2 × 每个综合性/设计性实验各1份
└── {课程名}_实验指导书.docx                   # Template 3 × 1
```

> **多班命名规范**：当 `classes` 数组含多个班级时（如信息可视化含影视班/游戏班），报告封面按班级分别生成，文件名追加 `_{classes[i].name 的班级简称}` 后缀，示例：`信息可视化设计_实验报告封面_实验1_影视班.docx`。班级简称从 `classes[i].name` 中提取（通常取最后 2-4 字）。实验指导书和认定表每门课程各1套，不区分班级。

### 4.4 报告封面字段责任分工说明

> 来源：Template_Exp_Report.docx 注释（作者：木慕，2026-01-31）

教务处学期末抽查明确表示封面中「项目名称」与指导书内容不符的情况会被记录。规定如下：

| 填写责任方 | 字段 | 生成策略 |
|-----------|------|---------|
| **程序预填**（任课教师发给学生前已填好）| 项目名称、学院名称、专业班级、授课教师、实验日期、实验地点、指导教师、一-三节正文 | 从 `course.yaml` / `context` 读取后写入 |
| **学生填写**（发给学生的模板保留空白）| 学生学号、学生姓名、同组人员（小组实验时）、四实验内容（步骤）、五实验结果分析 | **赋值为空字符串 `""`**，绝不从 context 读取任何数据 |
| **教师课后手填**（不由程序处理）| 评语、评分、指导教师签字 | **保留留白**（不替换占位符）|

## 5. 生成器注意事项 — 已知陷阱 (Lessons Learned)

> 综合过往项目 `2025-2026-1 _Lab_Report` 实战经验 + 通用 docx 引擎陷阱清单 + 模板注释分析（2026-02-24）。

| # | 问题现象 | 根因 | 修复方案 |
|:--:|---------|------|----------|
| 1 | 勾选框替换后格式出现乱码 | 括号 `（   ）` 被 Word 拆分到多个 `<w:r>` | 先调用 `_merge_runs()` 合并再替换 |
| 2 | 指导书各实验节内容互相污染 | `deepcopy` 后 cell 保留模板示例段落，`set_run_text()` 只修改第一段 | 赋值前调用 `_clean_cell_paragraphs(tc)` 清除多余段落 |
| 3 | 多行文本（目标/方法）只显示第一行 | 直接写入 `<w:t>` 中的 `\n`，XML 解析器忽略换行 | 用 `fill_multiline()` 为每行单独克隆 `<w:p>` 段落，保留原始 `pPr` |
| 4 | 封面学期字段显示原始编码 `2025-2026-2` | 编码字段未解析直接传入 | `prepare_experiment_context()` 中用正则 `(\d{4})-(\d{4})-([12])` 解析为中文学期格式 |
| 5 | `experiments` 为空时脚本崩溃 | 未做 None / 空列表判断 | 入口处 `if not context.get('experiments'): return` |
| 6 | 认定表生成时类型匹配错误 | 字段值含全/半角差异（如 `综合性` vs `综合性`） | 统一 strip + 全角比较；或在 Schema 层枚举校验 |
| 7 | Guide 模板大文件处理超时 | 含大量内嵌图片（~2.1 MB），ZIP 读写慢 | 已知现象，非 Bug；必要时提示用户耐心等待；禁止使用内存映射绕开 ZIP 方案 |
| 8 | 学号/姓名字段意外填写 | 报告封面被错误传入学生数据 | 空白分发模式下，学号/姓名/日期字段必须赋值为空字符串 `""` 而非从 context 读取 |
| 9 | Word 打开报「unreadable content」 | 插入了孤立的 `<w:commentReference>`（无配对的 `commentRangeStart/End`）| OOXML 规范要求三者必须同时存在且在同一 `<w:p>` 内；添加注释时要么三者全加，要么全不加 |
| 10 | 认定表「开课专业」与「学院」混淆 | 认定表没有「学院」字段，只有「开课专业」| 认定表不填 `department`，仅填 `major`；注释特别说明「不要带上年级」 |
| 11 | 实验三/综合项目意外生成「六、思考题」节 | 克隆首节结构时未过滤思考题节 | 克隆前检查 `exp.id`，仅 id ≤ 2 时包含「六、思考题」节；综合项目亦无此节 |
| 12 | 认定表日期写死或与开学时间不符 | 日期未从 semester 推断 | 从 `semester` 字段解析第一周周一日期，格式为「XXXX年X月X日」|
| 13 | 多班课程报告封面日期写错 | 旧版使用固定等间距公式 `(exp_id-1)*2` 推算实验周次，完全忽略 `calendar[].exp_id` 实际关联 | 从 `calendar` 查找 `exp_id` 匹配条目的 calendar 序号，通过 `week_range` + `excluded_weeks` 映射为班级实际教学周，再取该周上课日期。每班各自生成一套报告封面（2026-02-28 修复） |
| 14 | 认定表漏生成（只生成实验4的）| 触发逻辑误以为仅期末综合项目触发 | 遍历全部 `experiments`，凡 `type in ['综合性','设计性']` 均触发，每实验各1份 |
