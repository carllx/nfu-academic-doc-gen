# 教学大纲编写指南 (Spec_Syllabus)

> 来源：25-26-2 教学材料注意事项 - 第1节
> 参考：Syllabus.docx 批注归纳（37条，截止 2026-02-22）
> 整合：note1.md OBE 规范 · ADR 004/005

## 1. 核心规则 (Core Rules)
*   **一致性**：必须与 2025 年普本人才培养方案（观测点）完全一致。
*   **负责人制度**：课程负责人全权审核，任课教师签字仅由负责人签署。
*   **一课一纲**：不同年级/层次（如普本/专升本）即使同名，也需使用不同大纲。
*   **标题匹配**：一级、二级标题应与教材对应，匹配度 ≥ 80%。
*   **前沿融入**：必须融入人工智能、前沿技术，体现 OBE 成果导向理念。

## 2. 格式规范 (Format)
*   **字体与段落**：
    *   **封面/正文**：宋体小四。
    *   **标题**：各级标题相应加粗。
    *   **缩进**：首行缩进 2 字符。
    *   **行距**：固定值 20 磅。

## 3. 课程基本信息 (Course Info)
*   **名称与代码**：
    *   名称必须使用学校规范名称，与人才培养方案、教务系统完全一致。
    *   代码格式示例：`CNFU000001`。
*   **开课年级**：格式为 `202X级`。
*   **学时**：
    *   填写**总学时** (`hours.total`)，无需在基本信息处分开写理论/实践学时（但在章节内容中需拆分）。
*   **课程性质** (`nature`)：
    *   必须对应毕业学分结构的**二级标签**。
    *   映射关系：**专业必修课** 包含但不限于专业核心课、专业方向课、就业指导、劳动教育等。
    *   **格式**：使用勾选框形式，**不换行**，所有选项在同一行内显示。
    *   **选项（规范 5 类）**：☐专业必修课 ☐专业选修课 ☐公共必修课 ☐公共选修课 ☐成长必修课
    *   **填写逻辑**：根据 `course.nature` 自动判定勾选状态。
    *   **`course.yaml` 合法值**（实际位于 `course_meta.yaml`）：`专业必修课` / `专业选修课` / `公共必修课` / `公共选修课` / `成长必修课`
*   **先修课程** (`prerequisites`)：
    *   具有严格逻辑关系的课程（未学习先修课程则无法学习该课程）。
    *   若无，必须填写"无"。
*   **课程简介** (`introduction`)：
    *   **内容** (`content`)：简要描述本课程对毕业要求的支撑作用以及主要教学内容。
    *   **教学设计思路** (`design`)：阐述课程的教学设计理念与实施路径。
*   **课程建设情况**：如省级一流课程、在线开放课程等。如没有，写"无"。

## 4. 课程目标 (Objectives)
*   **编写视角**：必须从**学生 (Student-Centered)** 角度表述（"解释/分析/应用/评价/创建..."）。
*   **OBE 行为动词规范**（来源: note1.md §6）：
    *   ❌ **禁用**：了解、熟悉、理解、掌握（模糊，不可测）。
    *   ✅ **推荐**：列举、解释、应用、分析、综合、评价、创建、执行、描述、阐述、构建、演示、运用、比较、验证、评定。
    *   **结构**：`行为动词` + `学习内容/对象` + `表现标准/条件`
*   **三维矩阵**：
    *   **知识目标** (`knowledge`)：解释、阐述、列举、描述、比较。
    *   **能力目标** (`ability`)：分析、评价、创造、应用、执行、构建。
    *   **素质目标** (`quality`)：价值观、职业素养、规范意识。
*   **毕业要求映射**：
    *   必须明确对应人才培养方案中的**毕业要求**及其**观测点**。
    *   **禁止瞎填**：需核对人培结构化数据（见 `00_Data_Context/graduation_requirements.yaml` 观测点命名 + `training_plan_2025.yaml` 课程支撑矩阵），引用正确的序号和内容。修改后须运行 `audit_course_data.py` 确认无 `[TRAINING_PLAN]` 级别错误。

## 5. 教学内容与要求 (Content)
*   **章节结构**：
    *   **主要知识点**：按 `1.1`, `1.2` 编号罗列。
    *   **参考教材**（Phase 2d 新增）：知识点条目之后、教学要求之前，自动追加一行 `参考教材：` 引用文本。数据源为 `calendar[].readings`，由 `resolve_chapter_titles()` 解析为带章节标题的全称（full 格式）。无 `readings` 或无 `toc` 时不输出。使用知识点条目模板样式（非加粗），不带编号。
    *   **重难点**：明确指出"重点"、"难点"，需对应课程目标。
    *   **课程思政**：每章必须有"课程思政融入点"。
    *   **教学方法**：如案例演示、小组讨论等。
*   **学时分配**：
    *   每章需明确 **理论学时** (`hours_theory`) 和 **实践学时** (`hours_practice`)。
    *   学时均为**整数**。
    *   理论课程的实践学时不能超过 18（如 26:10 是上限）。
    *   **校验**：所有章节学时之和必须等于课程总学时 (`hours.total`)。
    *   学时变化需同步修改实验指导书。

## 6. 考核与成绩 (Assessment)

### 6.1 总评构成
*   **总评** = 平时成绩 + 课程考核(期末) = **100**
*   **2024级及以上**：过程性 (50-70%) + 期末 (30-50%)。考勤约占过程性评价的 1/3。
*   **2025级及以下**：必修课过程性 ≤ 30%，其他课程 ≤ 50%。
*   **卷面限制**（仅2025级及以下）：卷面成绩 < 50分，总评即为卷面成绩。
*   **告知义务**：必须在第一次课告知学生成绩构成、比例和评定依据。

### 6.2 过程性考核(平时成绩)结构

| 评分构成 | 子项 | 占比 | 分值 | 说明 |
|----------|------|------|------|------|
| **平时成绩** (XX%) | 1. 考勤 | 10% (根据要求) | 100 | 旷课扣20分/迟到扣10分/请假扣5分/公假不扣 |
| | 2. 章节测试1 | XX% | 100 | 名称格式：`章节测试N` 或 `命题测试N` 对应 `实验N`|
| | 3. 章节测试2 | XX% | 100 | 同上 |
| | 4. 章节测试3 | XX% | 100 | 同上 |

> [!IMPORTANT]
> **ADR 005 命名规范**：
> - `name` 格式 = `章节测试N` 或 `命题测试N`（**禁止括号**、**禁止内嵌实验名称**）。
> - `desc` 必备结构 = `对应实验N「实验名称」。考核要求：……`（先声明外键关联，再给出具体考核指标）。
> - 考核要求需契合教学大纲的课程目标、能力目标、素质目标和授课内容。

*   **项目数**：不少于 3 种（调研报告、实验、测试、PPT演示、论文、作业等）。
*   **限制**：不建议单独用讨论、提问、汇报作为独立评分项，建议融合在其他训练中。
*   **分值规则**：分值列统一为卷面满分 **100 分制**，各子项权重通过占比百分比体现，不做折合。
*   **模板预留行机制**：大纲生成器会自动扫描表格中从 R5 到底部所有包含 `XXXX` 的实验/考核预留行。当填入配置的 `normal_items` 后，多余的 `XXXX` 空余行会被自动截断和删除，无需在模板中手动增减行。

### 6.3 期末考核(课程考核成绩)结构

| 评分构成 | 形式 | 分值 | 说明 |
|----------|------|------|------|
| **课程考核成绩** (XX%) | 考查/考试 | 100 | 分值统一为卷面满分100分制，占比通过百分比体现 |

*   **必修课**：必须"考试"。
*   **选修课**：考试或考查。
*   **考核要求**：教师根据课程实际考核需求填写（需契合教学大纲的课程目标、能力目标、素质目标和授课内容）。
*   **`exams` 必填**（ADR 004）：即使考查课也必须填写 `exams` 节点，否则模板渲染报错。

### 6.4 取消考核资格条件（保留原句）
*   学期请假时间累计达总天数 1/3
*   单门课程缺课达总学时 1/3
*   欠交作业达总次数 1/3 或不及格达 1/2

## 7. 教材与参考书 (Textbooks)

> 详见 Spec_Global.md §5.6 课程资料规范

### 7.1 选用教材
*   涉及**马工程教材**的课程**必须**使用马工程教材。
*   选用**近三年出版**教材，鼓励选用国家级、省级规划教材。
*   教材**一本**（主教材）。
*   引用格式（专著 [M]）：`作者. 书名[M]. 出版地: 出版社, 出版年份.`
    *   示例：`斯蒂芬 A.罗斯. 公司理财[M]. 北京: 机械工业出版社, 2017.`
*   **教材匹配**：教学内容体系与教材章节体系保持一致，一级/二级标题匹配度 ≥ **80%**。

### 7.2 参考书目与文献
*   参考书目**多本**。
*   书籍引用格式（专著 [M]）：同 §7.1 教材格式。
*   期刊文章引用格式（期刊 [J]）：`作者. 题名[J]. 刊名, 年, 卷(期).`
    *   示例：`张三. 交互设计方法研究[J]. 设计学报, 2024, 10(2).`

## 8. 课程网站等支持条件
*   **数据字段**：`course.resources_url`
*   填写课程网站、在线教学平台（如省级一流课程网站、慕课平台等）等支持条件。
*   如没有，写 **"无"**。

## 9. 数据与模板管理 (Data & Template Management)
*   **Schema 映射**：
    *   课程性质 -> `course.nature` (RichText 自动勾选，**不换行**)
    *   课程简介 -> `course.introduction.{content, design}` (RichText，`\n` → `\a`)
    *   学时 -> `course.hours.total`, `calendar[].hours_theory/hours_practice`
    *   目标矩阵 -> `course.objectives.{knowledge, ability, quality}`
    *   考核比例 -> `assessment_methods.{normal_score_ratio, final_score_ratio, normal_items}`
    *   期末考核 -> `exams` (**必填**，ADR 004)
    *   课程网站 -> `course.resources_url`
*   **模板验证**：
    ```bash
    python scripts/verify_syllabus.py "Output/2025-2026-2_教学大纲_xxx.docx"
    ```

## 10. 代码生成方案 (Generation Architecture)

> **当前方案：方案 A — XML 直接操作**（已弃用 docxtpl）

*   **流程**：`Syllabus.docx`(ZIP) → 解压 `document.xml` → lxml 精确替换 XXXX → 重新打包 ZIP
*   **优势**：100% 保留原始格式（vMerge、spacing、字体全部不变），无 docxtpl 渲染破坏风险
*   **核心引擎**：`scripts/gen_syllabus_xml.py`
*   **章节内容**：采用**模板深克隆**（方案②），从原始模板提取段落组，`copy.deepcopy()` 后仅替换 XXXX 文本
*   **模板即 SSOT**：所有格式属性（字号、字体、粗体、行距、缩进）完全由 `Syllabus.docx` 原始 XML 决定，代码不构造任何格式
*   **图片格式**：模板 logo 已从 VML (`v:shape`) 迁移至 DrawingML (`wp:inline`)，确保 LibreOffice PDF 转换时宽高比正确 (2026-02-28)
