# 规则：数据安全防御 (Data Safety & Defensive Coding)

> **核心原则**：任何字段都可能缺失、类型异构或格式非预期——代码必须防御性取值。

## §1 字段缺失与类型兼容

- **字段缺失不崩溃**：用 `data.get('key', {})` 链式取值，非标准课程缺少字段时优雅跳过
- **类型异构兼容**：`hours` 可能是 `dict` 或 `str`，`experiments` 可能不存在，代码须 type-check
- **防御性取值**：所有从 `course.yaml` 读取的字段一律用 `.get()` + 默认值
- **Spec 有/无数据规则对齐**：同一字段的"有数据时"和"无数据时"两套输出规则必须覆盖相同维度集合

## §2 编码与格式化

- **编码化字段须自动解析**：如 `semester: '2025-2026-2'` 必须在 `prepare_*_context` 阶段用正则解析为 `2025～2026学年第二学期`
- **显示格式字段须格式化函数**：复合格式字段（如 `team`）必须通过专用函数处理，兼容所有合法数据类型
- **格式化函数不跨字段借用数据**：`_format_team()` 仅使用目标字段自身的数据，不从其他语义无关字段"补全"
- **文本拼接防标点叠加**：拼接 `desc` 时先 `.rstrip('。.')` 再追加分隔符

## §3 数据引用一致性

- **数据引用类型一致**：代码按 index 匹配的字段，YAML 中必须存储 index 而非文字描述
- **Demo 数据与 Spec 字段名一致**：`course.yaml` 的字段名必须与 Spec 完全一致（如 `stage` 不可写成 `phase`）
- **加载即使用**：加载了的外部数据必须参与实际逻辑，不得"加载但不用"
- **死代码及时清理**：已迁移的旧函数不以"保留以备回退"留存，Git 历史即回退措施

## §4 Schema-Code 原子同步

- 生成器新增对 YAML 字段的依赖时，必须**同一会话内**同步更新 `course_schema.py` 和 `Data_Dictionary.md`
- `CalendarWeek` 的 `extra='allow'` 会让未声明字段静默通过校验——警惕遗漏
- **跨层类型对齐**：字段类型在三层（Schema / Spec / Data_Dictionary）必须一致

## §5 代码规范

- **深克隆优于构造**：保留格式场景用 `copy.deepcopy()` 而非手动构造 XML
- **通用引擎统一调用**：所有生成器通过 `docx_engine.render_docx()` 入口，不直接操作 ZIP
- **验证驱动**：每次修改后运行验证脚本确认无回归
- **第三方库须显式导入**：新用到的库须在文件顶部 `import`，宿主模块的库不自动继承
- **修改计划必须先查 Spec**：制定代码干预方案前先查阅 `Spec_*.md` 约束

## §6 模板数据规范

- **学生手写字段保留**：报告封面的学号/姓名/日期等保留 XXXX（空白）
- **勾选框文本化**：认定表项目类型勾选用 `（ √ ）` 文本替换
- **模板 vs 样本**：模板（含 XXXX）与参考样本须严格区分命名
- **模板示例数据≠字段语义**：不可从官方模板示例反推字段语义，须通过逻辑推断 + 官方说明确认

## §7 OBE 合规

- **OBE 行为动词合规**：`objectives.desc` 禁止以了解/熟悉/理解/掌握开头，推荐 Bloom 分类法动词
- **`supported_objectives` 全覆盖**：全课程周次汇总后须覆盖所有目标条目，无孤岛目标
- **`teaching_requirements` 结构化优先**：优先使用 `{knowledge, ability, quality, method}` 结构化格式
- **WYSIWYG 禁止自动编号**：生成器严禁对 `topic`/`content` 自动拼接编号

## §8 学时与分值

- **物理课时红线**：单项 > 5 即 `[CRITICAL]`；路演周强制 `hours_theory == 0`
- **分值比例动态化**：模板硬编码分值仅为占位，须依业务基准线动态计算
- **教学环节 stage 分类校验**：理论类 stage 折算须与 `hours_theory` 对应；第一周禁止 `stage=复习`

## §9 多班级与偏移映射

- **calendar 数据不可跨班复制**（SSOT 破坏）：通过 `week_range` 偏移映射 + `excluded_weeks` + `session_time_overrides` 声明差异
- **offset 映射须感知中间空洞**：映射前构建 `skip_weeks` 集合，跳过停课位置
- **班级差异声明优于数据冗余**

## ❌ 禁止行为

- ❌ 不用 `.get()` 直接取字段值
- ❌ 格式化函数跨字段借用数据
- ❌ 在 YAML 中用文字描述替代 index
- ❌ 新增字段依赖而不同步更新 Schema
- ❌ 为第二班复制全量 calendar 条目
