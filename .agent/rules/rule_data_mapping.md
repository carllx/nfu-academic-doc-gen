# 规则：数据映射与格式偏好 (Data Mapping & Output Preferences)

> **核心原则**：数据内聚于 YAML，格式忠于官方模板案例。

## §1 数据映射偏好

- **方案C 偏移映射**：多班级周次不同时不复制 calendar，由生成器根据 `week_range` 偏移映射。自动跳过 `excluded_weeks` 和节假日周，calendar 条目顺延
- **分段排课内聚**：排课差异通过 `schedule_segments` 在 class 级别声明
- **数据内聚于 YAML**：小型数据（如 `support_level`）内嵌到 `course.yaml`，不拆为独立文件
- **数组优于扁平**：存在一对多映射可能性时优先数组结构（已应用于 `objectives[].mappings[]`）
- **数据去重内嵌**：可内嵌的小型数据不应拆为独立文件

## §2 人培方案数据管理

- **人培方案为观测点唯一真值源**：`objectives` 中的编号必须与 `graduation_requirements_{version}.yaml`（SSOT）精确比对后填入
- **专升本映射规则**：专升本年级 + 2 = 对标普本年级（如 22级专升本→2024版人培、23级专升本→2025版人培）。专升本课程对标本科人培的毕业要求体系编写大纲
- **人培数据四层架构**：PDF 原始文档 → 清洗 Markdown → 结构化 YAML（观测点 SSOT + 课程矩阵 SSOT）
- **人培数据须结构化可机读**：审计脚本消费的数据必须提取为 YAML，不可依赖 Markdown
- **观测点命名须机器校验**：通过 `audit_course_data.py` 的 `[TRAINING_PLAN]` 级别自动校验
- **课程名矩阵一致性**：需在 `course.yaml` 顶部注释块记录人培官方课程名

## §3 字段语义约定

- **`teacher.team`**：协作教师，**不含主讲教师本人**。推荐 `[{name, title}]` 格式。无协作教师时填 `无`
- **`teacher.title`**：仅指学术职称（教授/副教授/讲师等），不可用岗位名称。无职称时填 `无`
- **首页格式**：课程团队 `姓名（职称）`，多人用 `；` 分隔；各类目标至少 3 条
- **跨项目字段协商**：需课程工作区提供新字段时先协商 SSOT 权属
- **MSG 消息须提供完整绝对路径**
- **跨项目任务须委托不直改**

- **`calendar[].readings`**：自由文本引用列表，由 `resolve_chapter_titles()` 解析为带标题的标准化引用。推荐使用以下可自动解析的格式：
  - 中文书名 + 中文章节号：`《书名》第一、二章` / `《书名》第1-3章`
  - 英文作者 + ChX：`Munzner Ch1` / `Author Ch3-5`
  - 字符串章节（无编号教材）：`《Refactoring UI》Section Group Name`
  - 综合复习（不可解析，保持原文）：`《书名》全书综合复习`
  - ❌ 禁止使用主题描述格式（如"交互定义与可用性目标篇"）——引擎无法解析

## §4 教案格式输出偏好

| 偏好 | 说明 |
|------|------|
| **中文数字标题** | 教案标题用「第一/二/三...次课教案」|
| **章节编号 WYSIWYG** | 由 `chapter_title` 字段直接提供，生成器不自动拼接 |
| **授课时间无日期** | 仅显示 `班名：第X周，周X N-M节` |
| **支撑目标含毕业要求** | `{序号}. {描述}。支撑课程{类别}目标（{idx}），对应毕业要求{编号}（观测点{point}）。` |
| **教学目标分类** | 知识目标/能力目标/素质目标/过程与方法四段 |
| **重难点段落分隔** | 重点和难点必须换行分隔 |
| **教学环节子节** | R10-R14 含【教学组织】【教学方法】【时间安排】【课堂思考】|
| **教学环节双源 Fallback** | 有 `steps[]` 按摘要填充，无数据自动生成占位框架 |
| **思政融入点** | 仅在 R12「新课讲授」行添加【课程思政融入点】|
| **模板评论即文档** | Word 评论视为填写规则第一手来源 |
