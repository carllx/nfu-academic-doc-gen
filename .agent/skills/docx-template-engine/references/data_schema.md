# 数据结构规范 (Data Schema)

在调用 `docx-template-filler` 暂存数据 (Staging Data) 以及传递数据给填充引擎时，必须严格遵守以下 JSON 结构与约束规范。

## 强约束与反 Null 规则

本规范实行“防御性”的强类型数据约束，旨在避免 Python 脚本在处理模板占位符时出现异常或输出带有 "None" 字样的文本。

1. **禁止 `nullable: true`**：
   在构建数据的 Schema 或推理阶段，任何字段均禁止声明为可空。
2. **禁止传 `null`**：
   在最终生成的 JSON 文件或对象中，禁止出现任何值为 `null` 的键值对。
3. **空字符串替代 (String constraints)**：
   如果预期数据为字符串类型但缺失，必须传入空字符串 `""`。
4. **数组循环与极端值约束 (Array Loops & Extremes constraints)**：
   - **空数组规范**：如果预期数据为列表类型但缺失或暂无内容，必须传入空数组 `[]`，严禁省略该键或传入 `null`。
   - **循环渲染与兜底**：为避免模板引擎在处理空数组循环时残留标签或崩溃，必须在模板端增加条件渲染（例如 `{% if items %}`）；若模板不可控，极端情况下应传入包含默认空字段的单元素数组（如 `[{"name": ""}]`）进行降级兜底。
   - **极端值处理**：对于超出常规大小的超大数组或过长文本等极端值，须在序列化和注入模板前进行合理截断限制，防止破坏文档布局或导致内存溢出。
5. **强类型约束**：
   数值、布尔值等其他类型必须保持一致，不得隐式混用。

## 基础 JSON 结构示例

以下是一个符合规范的数据示例：

```json
{
  "title": "项目总结报告",
  "author": "张三",
  "contact_info": {
    "phone": "13800138000",
    "email": "" 
  },
  "participants": [], 
  "tasks": [
    {
      "task_name": "需求分析",
      "status": "已完成",
      "remark": ""
    }
  ]
}
```

> **注意**：上述示例中，缺失的 `email` 和 `remark` 使用了空字符串 `""`，暂无人员的 `participants` 使用了空数组 `[]`，严格避免了 `null` 的出现。
