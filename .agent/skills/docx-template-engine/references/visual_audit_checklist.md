# 视觉审计检查清单

> 在执行 Agent 视觉审计时加载此文件。按优先级排序，逐项对照模板截图与输出截图。

## 目录

1. [致命缺陷（P0）](#p0-致命缺陷)
2. [严重问题（P1）](#p1-严重问题)
3. [一般问题（P2）](#p2-一般问题)
4. [微调建议（P3）](#p3-微调建议)

---

## P0 致命缺陷

必须立即修复，否则文档不可用于提交。

| # | 检查项 | 如何判断 | 修复方向 |
|:--|:--|:--|:--|
| P0-1 | **残留标签** | 页面上可见 `{{ }}`、`{% %}`、`{# #}` 原始文本 | 标签被拆分 → 运行 `repair_template.py`；context 缺字段 → 补充数据 |
| P0-2 | **页面缺失** | 输出文档比模板少页面 | 检查 `{% for %}` 循环是否生成了空列表；检查分页符是否被吞掉 |
| P0-3 | **整表消失** | 模板中有的表格在输出中完全不见 | 检查 `{% if %}` 条件是否错误地跳过了整个表格 |
| P0-4 | **乱码/编码错误** | 中文字符显示为方块或问号 | 检查 context 中的字符串编码；确认字体嵌入 |

## P1 严重问题

影响文档可读性或合规性。

| # | 检查项 | 如何判断 | 修复方向 |
|:--|:--|:--|:--|
| P1-1 | **字号不一致** | 填入的数据文字大小明显不同于模板中的其他文字 | 标签 rPr 被破坏 → 修改模板确保标签格式一致 |
| P1-2 | **表格溢出** | 单元格内容超出边框，文字被截断或重叠 | 长文本预截断；或修改模板单元格属性为自动换行 |
| P1-3 | **跨页断裂** | 表格在不应该断开的位置被分页 | 添加行高锁 `<w:cantSplit/>`；或缩短填入文本 |
| P1-4 | **对齐破坏** | 居中/右对齐的内容变成了左对齐 | 标签段落的 `<w:jc>` 被覆盖 → 检查模板标签格式 |
| P1-5 | **下划线丢失** | 模板中的填空下划线在数据填入后消失 | rPr 中的 `<w:u>` 属性被覆盖 → 模板修复 |
| P1-6 | **多余页面** | 输出比模板多出额外的空白页或重复页 | 检查 `{% for %}` 循环是否产生了多余迭代 |

## P2 一般问题

不影响内容正确性但影响美观。

| # | 检查项 | 如何判断 | 修复方向 |
|:--|:--|:--|:--|
| P2-1 | **行距变化** | 填入数据后某些行的间距明显不同 | 检查段落 `<w:spacing>` 是否被模板标签重置 |
| P2-2 | **字体切换** | 数据文字使用了与模板不同的字体 | 确认 context 未注入 RichText 对象覆盖字体 |
| P2-3 | **空白行残留** | `{% if %}` 跳过内容后留下了空行 | 使用 `{%p if %}` 或 `{%-` 语法消除空行 |
| P2-4 | **边框不完整** | 表格的部分边框线消失或变形 | 通常是 `{%tc %}` 标签影响了单元格属性 |

## P3 微调建议

可选优化项。

| # | 检查项 | 说明 |
|:--|:--|:--|
| P3-1 | **数据精度** | 日期格式是否符合要求（如"2024年9月"而非"2024-09"） |
| P3-2 | **标点规范** | 中文语境是否使用了中文标点 |
| P3-3 | **空值显示** | 确认空字段显示为空白而非"无"或"None"（除非业务要求） |

---

## 执行流程

```
1. 同时打开模板和输出的截图（同一页）
2. 按 P0 → P1 → P2 → P3 顺序检查
3. 发现任何 P0 问题 → 立即停止审计，先修复
4. 所有 P0/P1 通过后，记录 P2/P3 问题但不阻塞交付
5. 将发现的所有问题记录到审计报告
```

## 报告格式

```markdown
### 第 N 页审计

| 检查项 | 结果 | 说明 |
|:--|:--|:--|
| P0-1 残留标签 | ✅ 通过 | — |
| P1-1 字号一致性 | ⚠️ 问题 | "姓名"字段字号小于模板预设 |
| P1-2 表格溢出 | ✅ 通过 | — |
```
