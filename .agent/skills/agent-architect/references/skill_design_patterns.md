# Skill 设计模式

本文件合并了输出模式与工作流模式，作为创建 Skill 时的参考。

---

## 输出模板模式

### 严格模板（API 响应、数据格式等）

```markdown
## 报告结构

必须使用以下模板：

# [分析标题]

## 摘要
[核心发现的一段话概述]

## 关键发现
- 发现 1 + 支撑数据
- 发现 2 + 支撑数据

## 建议
1. 具体可执行建议
2. 具体可执行建议
```

### 灵活模板（允许适应性调整）

```markdown
## 报告结构

以下为默认格式，可按实际情况调整：

# [分析标题]

## 摘要
[概述]

## 关键发现
[根据实际发现调整章节]

## 建议
[根据场景定制]
```

---

## 示例模式

当输出质量依赖于范例时，提供输入/输出对：

```markdown
## 提交消息格式

按以下示例生成提交消息：

**示例 1:**
输入：添加了 JWT 用户认证
输出：
feat(auth): 实现基于 JWT 的认证

添加登录端点和令牌验证中间件

**示例 2:**
输入：修复了报告中日期显示错误
输出：
fix(reports): 修正时区转换中的日期格式化

在报告生成中统一使用 UTC 时间戳
```

示例比描述更能帮助理解预期的风格和详细程度。

## 工作流模式

### 顺序工作流

适用于多步操作，在 SKILL.md 开头提供流程概览：

```markdown
填充 PDF 表单的步骤：

1. 分析表单 (运行 analyze_form.py)
2. 创建字段映射 (编辑 fields.json)
3. 验证映射 (运行 validate_fields.py)
4. 填充表单 (运行 fill_form.py)
5. 检查输出 (运行 verify_output.py)
```

### 条件工作流

适用于有分支逻辑的任务：

```markdown
1. 判断修改类型：
   **创建新内容？** → 遵循下方"创建工作流"
   **编辑已有内容？** → 遵循下方"编辑工作流"

2. 创建工作流：[步骤]
3. 编辑工作流：[步骤]
```
