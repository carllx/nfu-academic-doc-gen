# 交接文档：课程端实验步骤强合规约束 (Hand-off to Course Agent)

## 1. 核心背景 (Background)
根据“实验规则铁律”底层的修改要求，教务系统（NFU 教务项目）目前已经全面升级了针对实验指导书的 Schema 校验。
由于最新规定：**生成的实验指导书必须自带实验步骤示范，且最终报告模板必须与之步骤一一对应**。

为了避免教务系统在生成大量 Word 模板时出现空白版面或导致督导审查退回，我们在教务系统生成端（`generate.py`）加装了**底层硬性阻断逻辑**。

## 2. 教务端的最新校验规则 (What we implemented)
- **强行阻断**：任何一个实验（无论设计性还是综合性），如果在其实际的增量配置文件（`practices/experiments/exp_{id}.yaml`）中，**既没有 `steps` 也没有 `guide_text`**，教务引擎将直接抛出 `[CRITICAL]` 级别的 Error 并 `sys.exit(1)`。
- **废弃类型拦截**：“验证性”与“演示性”已经被底层强制过滤或转换为“设计性”，以配合 40学时/60学时 的 3/4 次实验矩阵。

## 3. 对课程端 Agent 的行动诉求 (Action Required for Course Agent)
作为负责操作 `/Users/yamlam/Downloads/2025-2026-2 课程` 工作区的主控 Agent，你需要立即进行以下行为适配：

1. **源头拦截**：在生成或提示老师填写 `exp_{id}.yaml` 时，**必须强制要求老师提供 `steps` (步骤拆解) 及其对应的 `guide_text` (指导书正文)**。决不允许静默跳过。
2. **数据补全**：若你发现历史旧数据中存在部分实验缺失步骤的，请务必利用大语言模型的能力结合课程内容进行**草案补写**，或者明确中断流程要求人类用户提供内容，然后再执行教务生成。
3. **学时矩阵对齐**：请确保你们生成的 `course.yaml` 严格遵守以下要求：
   - 如果 `practice` 学时为 40，`experiments` 列表的长度**必须精准为 3**（2个设计性，1个综合性排在最后）。
   - 如果 `practice` 学时为 60，`experiments` 列表的长度**必须精准为 4**（3个设计性，1个综合性排在最后）。

## 4. 故障排查 (Troubleshooting)
如果你在调用 `generate.py` 时遇到如下报错：
```text
❌ [ERROR] 实验 'XXX' (ID: X) 缺失具体的实验步骤！
👉 教务新规要求指导书必须包含操作步骤占位。请前往课程工作区的 practices/experiments/exp_X.yaml 补充 `steps` 或 `guide_text`。
```
**不要尝试修改教务端的代码**，而是必须回到对应的 `exp_X.yaml` 补全字段并重试。
